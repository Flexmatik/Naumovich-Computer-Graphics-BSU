# Лабораторная работа 2: Обработка изображений

## Описание
Приложение для обработки изображений с использованием OpenCV. Поддерживает следующие операции:
- **Sharpen** — увеличение резкости изображения
- **Otsu Threshold** — автоматическая бинаризация по методу Оцу
- **Manual Threshold** — бинаризация с ручным выбором порога

## Требования
- C++17
- OpenCV 4.x
- CMake 3.10+
- Linux (Ubuntu/Debian)

## Установка зависимостей

### Ubuntu/Debian:
```bash
sudo apt-get update
sudo apt-get install -y libopencv-dev cmake build-essential
```

## Компиляция

1. Перейдите в директорию лабораторной работы:
```bash
cd /home/flexmatik/labsBsu/computer_graphics/Naumovich-Computer-Graphics-BSU/lab2
```

2. Создайте директорию для сборки:
```bash
mkdir -p build
cd build
```

3. Запустите CMake:
```bash
cmake ..
```

4. Соберите проект:
```bash
cmake --build .
```

## Запуск

### С изображением по умолчанию (test1.jpg):
```bash
./ImageLab
```

### С пользовательским изображением:
```bash
./ImageLab image.jpg
```

> **Важно:** Изображение должно находиться в родительской директории (`../image.jpg`)

## Управление

После запуска приложения нажимайте клавиши в окне результата:

| Клавиша | Действие |
|---------|----------|
| `s` | Увеличить резкость (Sharpen) |
| `o` | Применить порог Оцу (Otsu Threshold) |
| `t` | Применить ручной порог (Manual Threshold) |
| `r` | Сброс к исходному изображению (Reset) |
| `q` или `ESC` | Выход |

## Регулировка ручного порога

Используйте трекбар **"Threshold"** в окне результата для установки значения порога (0-255).
Затем нажмите `t` для применения.

## Решение проблем

**Ошибка: "OpenCV not found"**
```bash
cmake .. -DOpenCV_DIR=/usr/lib/x86_64-linux-gnu/cmake/opencv4
```

**Ошибка при загрузке изображения**
- Убедитесь, что файл `test1.jpg` находится в директории `lab2/`
- Или передайте полный путь: `./ImageLab ../path/to/image.jpg`
